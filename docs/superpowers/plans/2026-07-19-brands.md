# Brands ("Anime" tags) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Brand entity across the stack — admin CRUD + product tagging, storefront "Anime" navigation and brand filtering.

**Architecture:** Brand mirrors Category everywhere: Prisma model + NestJS module in the API, TanStack Query hooks + page in the admin, chips/menu in the storefront. Product.brandId is optional with `onDelete: SetNull`. The generated OpenAPI clients carry the contract to both frontends.

**Tech Stack:** NestJS + Prisma, React 19 + TanStack Query + React Hook Form/zod, Next.js 15.

**Spec:** `docs/superpowers/specs/2026-07-19-brands-design.md`

## Global Constraints

- Brand is optional on products; deleting a brand sets products' brandId to null (never blocks/cascades).
- Storefront label for the concept is **"Anime"**; admin label is **"Brands"**.
- Follow the existing Category implementations verbatim as templates (module layout, DTO validation incl. kebab-case slug regex, hooks, page structure).
- After API contract changes: `cd ecommerce-api && npm run openapi:emit`, then `npm run generate:api` in BOTH `ecommerce-admin` and `ecommerce-storefront` (committed generated code).
- Each task ends green: `npm test` + `npm run lint` in the touched app(s); commit per task.

### Task 1: API — Brand model, module, product integration

**Files:**
- Modify: `ecommerce-api/prisma/schema.prisma` (Brand model; Product.brand/brandId + index)
- Create: migration via `npx prisma migrate dev --name add_brand`
- Create: `ecommerce-api/src/brands/brands.module.ts`, `brands.controller.ts`, `brands.service.ts`, `brands.service.spec.ts`, `dto/{brand-response,create-brand,update-brand}.dto.ts` — each cloned from the `src/categories/` counterpart with Category→Brand renames
- Modify: `ecommerce-api/src/app.module.ts` (register BrandsModule)
- Modify: `ecommerce-api/src/products/dto/create-product.dto.ts` (optional nullable `brandId`), `dto/list-products.query.ts` (optional `brand` string), `dto/product-response.dto.ts` (BrandResponseDto on ProductResponseDto + `brandId` nullable on base), `products.controller.ts` (`@ApiQuery({ name: 'brand', ... })`), `products.service.ts` (include brand in all `include: { category: true }` sites → `{ category: true, brand: true }`; where-clause `...(brand ? { brand: { slug: brand } } : {})`; type `ProductWithRating` gains `brand?: Brand | null`)
- Test: `ecommerce-api/src/products/products.service.spec.ts` add brand-filter case

**Interfaces:**
- Produces (for Tasks 2–3 via generated clients): `brandsControllerFindAll/Create/Update/Remove`; `BrandResponseDto { id, name, slug, sortOrder, createdAt, updatedAt }`; `CreateProductDto.brandId?: string | null`; products list query param `brand`; `ProductResponseDto.brand?: BrandResponseDto`.

Schema addition:

```prisma
model Brand {
  id        String    @id @default(cuid())
  name      String
  slug      String    @unique
  sortOrder Int       @default(0)
  products  Product[]
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
}
```

Product additions: `brand Brand? @relation(fields: [brandId], references: [id], onDelete: SetNull)`, `brandId String?`, `@@index([brandId])`.

CreateProductDto addition:

```ts
@IsOptional()
@ValidateIf((_, v) => v !== null)
@IsString()
brandId?: string | null;
```
